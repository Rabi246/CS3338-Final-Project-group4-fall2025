The frontend serves as the user interface of the system. It allows users to upload files, view results, and interact with the application.

\subsection{Database}

\section{Docker Container Architecture}

The system is deployed using Docker Compose, which orchestrates three primary containers:

\begin{itemize}
    \item \textbf{Frontend Container} – React-based UI, exposed on port 3000.
    \item \textbf{Backend Container} – Node/Express API, exposed on port 5000.
    \item \textbf{Database Container} – PostgreSQL instance, internal port 5432.
\end{itemize}

Each container runs in an isolated environment and communicates over a shared Docker network created by Docker Compose.

\section{Container Dependencies and Required Libraries}

This section lists all packages and dependencies required inside each Docker container to run the system.

\subsection{Frontend Container Dependencies}

\begin{itemize}
    \item \textbf{Base Image:} node:20-alpine
    \item \textbf{JavaScript Libraries:}
    \begin{itemize}
        \item react
        \item react-dom
        \item react-router-dom
        \item vite or react-scripts
    \end{itemize}
\end{itemize}

\subsection{Backend Container Dependencies}

\begin{itemize}
    \item \textbf{Base Image:} node:20-alpine
    \item \textbf{Required Node Packages:}
    \begin{itemize}
        \item express
        \item pg
        \item cors
        \item multer
        \item dotenv
        \item nodemon
    \end{itemize}
\end{itemize}

\subsection{Database Container Dependencies}

\begin{itemize}
    \item \textbf{Base Image:} postgres:16
    \item \textbf{Internal Tools:}
    \begin{itemize}
        \item psql command-line tools
        \item persistent Docker volume (\texttt{db-data})
    \end{itemize}
\end{itemize}

\subsection{Host Machine Requirements}

\begin{itemize}
    \item Docker Desktop installed
    \item Internet access to pull images
    \item Disk space for database volume and container layers
\end{itemize}
